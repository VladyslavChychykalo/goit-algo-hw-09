# goit-algo-hw-09

**Жадібний алгоритм**

- 0.00000191 секунд
- Швидкий і простий у реалізації.
- Працює швидко - O(n), де n - кількість номіналів.
- Не гарантує мінімальну кількість монет для нестандартних номіналів.

**Динамічне програмування**

- 0.00004101
- Завжди дає оптимальне рішення.
- Часова складність - O(n * sum).
- Може працювати повільніше для великих сум без оптимізацій.

Висновок
Для даної системи номіналів монет (Гарний набір, так як кожен менший номінал - кратний або майже кратний
більшому) [50, 25, 10, 5, 2, 1], жадібний алгоритм:

- Гарантує мінімальну кількість монет,
- Працює швидше,
- Простіший у реалізації.

Тому у цьому конкретному випадку жадібний алгоритм є кращим.
